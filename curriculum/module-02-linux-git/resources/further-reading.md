## Books

### Linux

**"The Linux Command Line" by William Shotts** ‚≠ê‚≠ê‚≠ê‚≠ê‚≠ê

- **Cost**: Free online, ~$30 print

## Tips for Learning

### For Linux

‚úÖ **Practice in Safe Environment**
- Use VMs or containers
- Break things intentionally
- Learn from mistakes

‚úÖ **Use Man Pages**
```bash
man command        # Read full manual
man -k keyword     # Search for keyword
```

‚úÖ **Build Projects**
- Automate your daily tasks
- Create system monitoring scripts
- Set up a web server

‚ùå **Avoid**
- Copy-pasting without understanding
- Skipping error messages
- Forgetting to back up before experiments

### For Git

‚úÖ **Commit Often**
- Small, frequent commits
- Better than large, rare commits
- Easy to review and revert

‚úÖ **Write Good Commit Messages**
- Follow conventional commits
- Explain WHY, not WHAT
- Future you will thank you

‚úÖ **Practice Workflows**
- Create feature branches
- Do code reviews (even alone)
- Use pull requests

‚ùå **Avoid**
- Committing secrets
- Vague commit messages
- Working only on main branch

## Project Ideas for Practice

### Linux Projects

1. **Automated Backup System**
   - Back up important directories
   - Compress and encrypt
   - Keep rotation policy
   - Send email notifications

2. **System Monitoring Dashboard**
   - Monitor CPU, memory, disk
   - Alert on thresholds
   - Generate daily reports
   - Create web dashboard (bonus)

3. **Log Analyzer**
   - Parse server logs
   - Identify errors and patterns
   - Generate statistics
   - Create visualizations

4. **Server Setup Automation**
   - Install and configure services
   - Harden security
   - Set up firewall
   - Configure users and permissions

### Git Projects

1. **Open Source Contribution**
   - Find beginner-friendly issues
   - Fork, clone, branch
   - Make contribution
   - Submit pull request

2. **Personal Project with Full Workflow**
   - Use feature branches
   - Write conventional commits
   - Create pull requests
   - Use GitHub Actions

3. **Git Workflow Documentation**
   - Document your team's workflow
   - Create diagrams
   - Write best practices
   - Include examples
